零時政府媒體政策

邀約管道

演講與採訪邀約由 g0v-talks 群組擔任窗口，聯繫方式為 g0v-talks@googlegroups.com

  • g0v-talks 是一個任務性質的群組，工作內容為釐清演講與採訪邀約需求，轉發給適當的專案貢獻者。群組設定為非公開，目的是提供外界一個和社群私下聯繫的管道，處理牽涉到個人資料、個人隱私等不便在公開場合討論的事務。群組成立之初是為了處理演講邀約，後來隨著其他類型的邀約出現，演變為處理所有需要對外出席或發言的邀約，包括：演講、採訪、諮詢等。
  • g0v-talks 群組由 clkao 發起，目前成員包括：(待補)
    • 如果您不希望信件內容被以上群組成員看到，可單獨寄信給邀約對象。
  • g0v-talks 群組規則為：
    • 成員條件：有給過以 g0v 為主題的 talk，或被認為對 g0v 有足夠的了解，並能給 talk 的人，即可被提議加入
    • 成員加入與退出：提議 72hr 內，三人以上同意，不超過半數成員反對，即可加入。超過本 mailing list 成員的半數人反對，則可否決新邀請名單。超過 2/3 成員連署，則可取消既有成員資格
    • 演講回覆時程：邀約收到三天內，若無人能接受邀約，則視同無法進行，成員有義務通知邀請人無法進行
    • 演講成果：投影片及錄影 (若有) 以相容於 CC BY-SA 4.0 的授權 (含 CC0、CC-BY) 釋出 
    • 演講費用及稅務：由實際演講人領取/負責，可自由選擇捐贈贊助黑客松
    • 本規則修訂：需經提出後 72hr 期間，三人以上同意，不超過半數成員反對

對外發言原則

  • 維持主體性：g0v 和其他媒體站在對等的地位，可當供稿主體，但不做為受訪客體
    • 這是為了避免傳統媒體形式中常見的公關問題，如被記者斷章取義、扭曲原意、錯置重點等
  • 維持分散性：供稿內容由各別作者 / 講者以 cc 釋出，每個人為自己的言論負責，彼此不互相代表
    • 這是為了維護開源專案參與者的言論自主，確保個人意見不會被他人扭曲
  • 違反主體性或分散性原則的媒體活動：
    • 採訪主軸是社群或專案事務 - 在開源模式尚未廣為人知的階段，不建議以專案參與者名義列席或署名，以避免替社群帶來額外的公關問題。可改以非營利組織或公司行號名義列席或署名
    • 採訪主軸不是社群或專案事務 - 可以與社群或專案無關的名義列席或署名
  • 沒討論過、尚未形成共識的媒體活動：以公開 hackpad 共筆或公開 irc 的形式，在時間充裕、過程透明的狀態下討論，尋求社群共識後，再執行

供稿名義

能夠正確描述供稿者身份的詞彙

  • ____專案協調人
  • ____專案貢獻者
  • ____專案參與者

也常用但可能有點問題的詞彙

  • ____專案窗口 - 窗口通常是所有參與者可看見的 mailing list address，而不是由特定的人把持

不符合事實的詞彙

  • g0v 成員 / ____專案成員 - 非組織無法界定成員
  • g0v 代表 / ____專案代表 - 誰都不能夠代表別人

供稿形式

包括

  • 採訪共筆：社群參與者在公開 hackpad 上自主回答問題。例： (不勝枚舉)
  • 參與黑客松：記者報名參加黑客松，以參與者的身份分享活動見聞。例：新新聞、天下
  • 深度訪談：訪談前雙方需充份討論準備，並自己設定節奏。例：深音
  • 演講：由講者和邀約者討論主題範圍。例： (也是不勝枚舉)
  • 直播：社群活動發起人自主開的網路直播。例：歷次黑客松

不包括

  • 新聞稿：新聞稿是傳統的媒體形式，帶有「代表團體宣誓或背書」的文化意涵，不符合分散性原則。若新聞稿主軸與社群或專案有關，在開源模式尚未廣為人知的階段，不建議以專案參與者名義在新聞稿中署名，可改以非營利組織或公司行號名義署名
    • 雖不建議署名，仍建議專案參與者以個人身份在發稿前接受諮詢，以避免其他團體在新聞稿內對社群或專案有錯誤描述
  • 記者會：記者會是傳統的媒體形式，帶有「代表團體宣誓或背書，並請記者代為宣傳」的文化意涵，不符合主體性、分散性原則。若記者會主軸與社群或專案有關，在開源模式尚未廣為人知的階段，不建議以專案參與者名義列席，可改以非營利組織或公司行號名義列席
    • 雖不建議列席，仍建議專案參與者以個人身份出席，在觀眾席上即時彙傳或糾正，以避免其他團體在記者會上對社群或專案有錯誤描述

供稿內容

以發揚「開源協作 x 公民參與 x 行動主義」核心精神、增進實際成果產出為原則，包括

  • 推廣開源文化
  • 吸引 contributor
  • 影響其他 NGO 協作模式

不包括

  • 支持或呼籲上述核心精神以外的議題 - 只要做事方法相同，立場不同的人也能一起打造工具；除了做事方法以外的特定立場，則通常會排斥立場不同的人，使潛在 contributor 減少。所以，如果在開源協作的方法論之外，有想支持的特定議題，可改以非營利組織或公司行號名義發表，與社群或專案無關
