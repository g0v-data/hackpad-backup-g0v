UX/UI 支援

Ui Gathering台灣使用者經驗設計協會想透過實作來支援 g0v 專案，同時推廣 UX、也讓相關學生累積實際經驗。之後可能會先有個設計松來聊一下協作方式。

UiGathering.org 介紹

各位好，我們是台灣使用者經驗設計協會(www.uigathering.org)。協會從2005開始辦小型活動推廣UI/UX的專業，到2012正式成立協會。但近年來，我們也憂心「使用者經驗/UX」這一詞變成蛋塔，只是拿來消費的名詞(其實已經看到有很多單位這樣做了)。我們希望今年不只是停留在推廣UX，也希望透過實際參與來深化。所以今年想要搭起UX設計師/研究者/學生，與其他社會公益行動橋梁。協會也希望安排資深/資淺不同UX的組合，在過程中作我們的推廣與培育工作。我們也是今年開始這樣的嘗試，還請各位多多指教!!!!  (阿亮 xxc@uigathering)

國外有個 Open Usability 的計畫，連結使用者經驗與開源軟體兩個專業社群。使用者經驗專家（包含學生）可以做一些沒有商業包袱的專案，開源軟體社群又可以獲得一些使用者介面與經驗改善的協助。或許我們可以嘗試建構類似的平台。

專案簽名

有需要 Ui 支援的專案請先簽名，列出目前成果，以及希望的協助：

  • leve1up http://g0v.github.io/leve1up
    • 給剛接觸 g0v 的一般人了解 g0v 在做什麼、如何參與的一份互動文件
    • 以 checklist 的形式組成，避免落落長的文章讓人迷路，也能紀錄進度。
    • 有基本的 feedback 功能，頁面右方內嵌了一個 firepad （類似 hackpad 的即時協作工具）
    • 從 markdown (https://raw.github.com/g0v/leve1up/master/g0v.md) 透過自製小工具 (https://github.com/poga/moltenpad) 產生的
    • 希望的協助：
      • 想做什麼都可以 XD
      • 版面設計：此頁面會需要嵌入至其他網頁中。（ex. http://g0v.tw/join.html） 目前的作法是直接將右方 firepad 移除，讓頁面在較窄的空間中也不要太難看。
      • 如何將即時編輯/回應系統更好的整合進文件中：目前右方的 firepad 與內文沒有任何連接關係，實用價值不高。
  • g0v.tw 社群官網 http://g0v.tw
    • 希望讓完全不知道 g0v 到深度貢獻者都可以從官網找到任意門。（目前各種 user 都會稍微迷路）
  • 人力資源部hack.g0v.tw/people
