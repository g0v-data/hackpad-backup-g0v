標籤

標籤為全系統共用

  • 任意指定
    • e.x : 統一+塑化劑 or 台北+都更
  • 使用者自己給自己加標籤, 因此事件與自己可以關聯
    • e.x: 台北, 鄉民
  • 使用者可用標籤選擇要關心的事件, 因此事件與自己可以關聯
    • e.x: 核四, 反核

標籤分兩種

  • 群組標籤: 屬於同一群組的標籤不可搭配使用. 例如: 地名
  • 獨立標籤

使用情境蒐集

非移動裝置 - 看到朋友發言或轉貼文章時, 不做任何評論

徐鳳年瀏覽Facebook中, 見一朋友分享連結，閱畢，略有所思，想做些事後追蹤，遂加進 In-Box。

steps:

  • 徐鳳年點擊icon or press a hotky
    • 一個頁面從瀏覽器右側向左滑出，網址title跟縮短後的網址會被自動填入，使用者可以寫評論，或加標籤。評論跟標籤都可在整理步驟再加工
    • mockup: https://moqups.com/hychen/2gtBEbns/
  • 徐鳳年繼續瀏覽Facebook

非移動裝置 - 看到朋友發言或轉貼文章時, 有做任何評論

徐鳳年瀏覽Facebook中, 見一朋友分享連結，閱畢，略有所思，在該討論串發表些想法, 發表的字串包含一個特殊的Tag, 使得這組討論串會被 In-Box。(let’s say subscribe)

  • 徐鳳年評論
    • 字串內包含 #inbox, 所以在回顧階段時，點選inbox標籤會看到這則發言,
    • 字串內包含 #核四, 所以點選inbox標籤也會看到這則發言
  • 徐鳳年繼續瀏覽Facebook

回顧

分為兩種步驟, /看/及/改/

非移動裝置 - 看

總統選舉快到了，吳素想查一下小彬彬這幾年到幹了什麼事，所以打開瀏覽器並點選小彬彬標籤，系統會顯示所有關聯到小賓賓標籤的事件跟評論 ，很快她就知道小彬彬的的專長是拆房子，並不適合總統這種需要具備默哀讀秒技能的職位。

非移動裝置 - 改

吳素星期日練完劍，卓磨著最近蒐集了不少事件跟想法，便打開瀏覽器, 針對每條事件跟評論做整理，有的加標籤，有的再多寫一筆想法。 特別是在霉體最近都在報些花花草草的消息時，她還能利用014這個標籤，透過她在社群網路上的朋友言論，繼續追蹤後續的變化。

特別狀況

小泥人的腦殘親人打算立委選舉時只因地緣關係而投給某個評價很差的候選人時，理由只是因為"再怎麼爛，他還是住這裡，會為這地方著想"這種原因時， 小泥人可拿出這幾年透過此系統蒐集的某個候選人資料做舉證，從而灌述她的腦殘親人，拯救這迷途羔羊。

Tag-based tracking

the idea is inspired by Emacs Org-mode. A way to orgalinze datas in plain-text.

open graph.

  • A tag is an identiy of an object or a attribute of an object
  • A tag can have properties 
    • what properties that a tag should has depends whihc kind it belongs to
  • A tag can associate to a/many objects
    • message, tag, a resouce on internet, etc.
    • tag can not associate to kinds.
  • A kind is a special tag like a container. there are 2 kind in this system:
    • set: can not have multiple tags which has the same name.
    • list: can have mutiple tags which has the same name.
  • A kind can only associate to tags
    • If a tag is a people’s name, we could add the tag into a kind "human" and add more properties like title, phone number, etc.
    • all tags in "human" kind might be associate to a existence system user.
  • the tag name convention is the same as twitter’s 
    • #{tag name}, @{people name}

As a diary

  • a diary entry is a message or a web resource which is associated to a tag "note" when it is created by a user
    • user could assoicate more tags on it in order to indicate the relations between the message on topics, events, people. the tags can also be used to indicate the attributes of the message.
  • we could think a tag works like a message channel and all users in the system shared tags created by each other so a user will see all messages relevent to the tag he is insteresting in.
    • a message can be a social network message also because the automatice social netowork messages assoication mechanism (describe later). user could be able to reply a social message in the system via proper social network website API.

social netowork messages assoication mechanism

nowdays, people discuss topics, news in social networks very often and there are too many social network websites. It bringns a problem that people is hard to track his comments and the feadbacks of his friends.

  • a tag in a social network is crersoponding to a tag in the system if their name are the same
  • a new message is assocaited to a tag "inbox" automatically when it is resolved by the system.
