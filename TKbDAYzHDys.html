g0v Dev Policy | 零時政府產品(?)開發政策

成為 g0v 專案的條件

  • 如果是軟體開發專案，必須採用任何一種 open source 授權
  • 如果是內容創作專案，必須採用任何一種 creative commons 授權
    • 為了盡量符合開源精神，工作檔案必須一併釋出

不能成為 g0v 專案的情況


成為 g0v 專案貢獻者的方式

  • 可以聯絡專案窗口，彼此討論有哪些適當的坑
  • 如果是軟體開發專案，也可以手刀衝 github 直接 fork 程式碼、提出 pull request
  • 如果是內容創作專案，也可以手刀衝 google drive 直接 fork 工作檔，建立自己的作品資料夾

外部商業 / 政府 / 非營利組織和 g0v 一起開發的方式

  • 政府
  • 非營利組織
  • 商業組織
