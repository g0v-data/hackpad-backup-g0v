User Story

列出各族群的目標、行為、相對應的 to-do 與 not-to-do

高度數位化，關心公眾事務的網民

  • 可能擁有一個以上社群網站帳號
  • 可能擁有一個以上行動裝置
  • 可能有資訊焦慮及網路成隱症
    • 例如爬個山也要帶個網路分享器
  • 不那麼熟悉政治及法律制度
  • 善於學習及使用數位科技工具解決問題
  • 認親都用ID而不是本名
  • 對於公眾議題較被動接收或接近漠視，主要使用本系統追蹤，並且用來分析各方論點及立場，以作為自己對議題，人，組織看法的依據

他可能會想要

  • 使用 API 製作客製化工具
    • 在網站上找到API使用手冊
  • 覺得本系統欠缺調較，想要參與本系統開發
    • 在網站上找到如何參與開發的資訊 (接到hackfold)
  • 跟中度數位化網民一樣使用本系統提供的功能

中度數位化，關心公眾事務的網民

  • 可能擁有一個以上社群網站帳號
  • 可能擁有一個以上行動裝置
  • 不那麼熟悉政治及法律制度
  • 對於公眾議題較被動接收或接近漠視，主要使用本系統追蹤，並且用來分析各方論點及立場，以作為自己對議題，人，組織看法的依據

他可能會想要

  • 紀錄當前瀏覽的網站及對其內容的想法
    • 點擊瀏覽器的某個按鈕，輸入標籤，或留言，送出後，他可以在系統的回顧功能中找到它
  • 腦海中閃過得念頭，想暫時記錄下來
    • 點擊瀏覽器的某個按鈕，留言，送出後，他可以在系統的回顧功能中找到它
  • 看看自己對某議題的看法，順便也看看別人的看法
    • 點擊回顧某議題頁面，瀏覽自己所有的評論，同時也可以看別人的評論
      • 應該:
        • 能明顯區別自己的評論跟別人的評論
        • 能設定黑名單忽略特定人士的評論
      • 不應該:
        • 自己不想讓人看到的評論被看到
        • 看到別人不想讓自己看到的評論
  • 分類自己蒐集的新聞或是討論串或是blog
    • 將這個新聞報導的事與人做關聯
      • 應該:
        • 系統提供政治人物清單作選擇
    • 指出這個新聞報導的事是某件事的起因
  • 可針對某議題的決策選擇支持或反對表達立場，讓關鍵人士聽見
    • 應該:
      • 可以轉成正式的連署
      • 系統提供文件範本可參考
    • 不應該:
      • 在這個階段蒐集身份證這類的重要個資
  • 產生客製化報表
  • 想關心近期最熱門議題
  • 想查看較冷門但重要的議題
    • 應該：
      • 可以按照網友評分檢索重要議題

社運人士

  • 對於公眾議題較主動參與，通常是行動發起人
  • 較熟系政治及法律制度
  • 可能會透過本系統的資料整理出說服文宣，尋求更多人支持他的立場
  • 可能會透過本系統進行串連，嘗試改變現狀，像是施壓代議士
  • 跟中度數位化網民一樣使用本系統提供的功能
  • Mina Liu: 不同世代的社運人士想法作法不同，年輕世代不一定想要跟政治人物接觸，老一輩則沒有能力製作懶人包，但也許會有其他的創意用途也不一定

他可能會想要

  • 將系統上民眾的聲音彙整轉交達給代議士或是政府決策者
  • 看看民眾/政治人物立場的分佈
  • 製作懶人包
  • 產生客製化報表

記者

  • 利用本系統蒐集報導素材
  • 跟中度數位化網民一樣使用本系統提供的功能
  • Mina Liu: 關於記者的部份，他們的確非常需要使用這個系統蒐集議題資料，但前提是他們知道有這個系統，而臉書是很好的傳播模式。因此首先要讓記者知道，然後記者透過一般媒體管道傳播給非臉書或尚未從臉書得知訊息的人。

他可能會想要利用這個系統

  • 看看民眾熱議的議題
  • 看看民眾/政治人物立場的分佈
  • 產生客製化報表

平常不關心公眾事務，投票前的選民

  • 對公眾議題參與較被動或近乎漠視
  • 投票通常是憑印象或是地緣關係，不太清楚候選人的豐功偉業
  • 想要利用這個系統了解候選人
  • Mina Liu: 關於平常不關心公眾事務的選民，我不太認為他們在投票前會主動去關心，而是在投票過程中接收到各種媒體訊息或口耳傳播影響到他們的投票意象。因此這些資訊在選舉過程中，絕對會影響到候選人的競選策略與文宣議題攻防，因此這些資訊將扮演重要的角色，候選人幕僚會是最大的讀者跟使用者。

他可能想要

  • 了解候選人在媒體上所有的發言或是相關新聞
  • 了解候選人在某些議題的包含過去及現在的立場
  • 了解候選人過去到現在是否有官司在身
  • 了解候選人跟某些公眾人物的關係
  • 產生客製化報表
