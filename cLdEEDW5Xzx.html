平面設計 / 視覺傳達

  • 概念
    • 第一本推薦看 寫給大家的平面設計書 The Non-designer’s Design Book，本書在二版時書名改成了 設計的上下左右
    • 第二本推薦看 設計的法則 Universal Principles of Design - 125個影響認知、增加美感，讓設計更好的關鍵法則【全新增訂版】
    • 中文教科書多半是垃圾
    • 其他的隨便 www
    • 設計和藝術是不一樣的東西，但常常被搞混，詳見 g0v XD talking pre hackath4n : 建築師觀點看設計之都 /Roy
    • https://hackdesign.org
  • 工具軟體
    • 推薦直接看官方的教學影片 Adobe TV，內容豐富又正確
    • 剛起步時需要有書帶頭的話，隨便挑一本就好，沒有特別推薦的
    • InDesign 入門請先看 zonble 經典文件 Adobe InDesign/InChinese 排版軟體基本操作
    • 繪圖軟體是為了美術工作者們設計的，如果不懂美術概念，就無法理解軟體的設計原則，學習時障礙也會特別多。
    • 大部分電腦書的「教學」方式是叫讀者照表操課，卻很少解釋 (或解釋不出來) 背後的概念，這種書看哪一家出的都沒差，所以說隨便挑一本就好。如果看到教實做的同時解釋背後概念、讓人恍然大悟的，那多半是值得購買的好書。

HTML / CSS / JS

  • 推薦上 codecademy (免費，課程內容錯誤較多) 和 code school (部分基礎課程免費，所有課程第一堂免費，課程內容正確性很高)
  • 上禮拜把web fundamentals 上完, chrome上有許多問題, 在Fx下面才能正確執行
  • codecademy 嗎 @@" \
  • 對! 我砍掉重練就是從codecademy開始XD
  • 查資料時看 MDN (Mozilla Developer Nerwork) 的 HTML element reference、CSS reference
  • 其他書籍都不怎麼推薦，如果需要書本的話…翻譯書優先吧 www
  • 除了codecademy 也可以看這邊 https://developer.mozilla.org/en-US/learn/
  • 這邊 JS 入門是看 JavaScript: The Good Parts 和  ECMA-262-3 in detail 配著看。後者適合本來寫 class-based OOP 者看看 JS 到底在搞什麼鬼。
  • 純 HTML / CSS 的練習，可以去 https://dash.generalassemb.ly/ ，對新手來說成就感挺高的！只是由於他的觀念教學零散地分布在各個章節，建議配合上面大家分享的資源，較能有系統地建立觀念。
  • HTML / CSS / JS 也可以考慮看 http://www.learnstreet.com/，和 codeacademy 一樣是用程式習題來教學，但 learnstreet 習題數比較少，重點大多有 cover 到。只是 JS 的部份有些缺漏。
  • CSS Diner 
  • CSS 來晚餐！用酷炫的食物動畫教你練習 CSS selector 。
  • Web Fundamentals
  • Google 教 Web ！

程式設計

  • 第一本推薦看 深入淺出程式設計 Head First Programming
  • 接下來也是推薦上 codecademy 和 code school
  • Ruby 人上完 codecademy 後可以先上 Ruby Monk - Ruby Primer (免費，解釋非常詳細) 
  • 覺得很抽象時推薦看 程式設計師的自我修練：連結、載入、程式庫 (開頭有基本的電腦運作原理)
  • 我覺得這是相當系統層面的書，不打算作系統的自學者讀這個有點過頭
  • 我的意見與你相反，個人覺得這是非常重要的groundwork，沒有系統整體的運作概念宛如樹不見林，會不知不覺在程式裡漏掉一些脈絡。只能說不用一開始就去當聖經來翻，等有Coding、Debug跟Tracing dirty code的經驗之後再來看會非常非常非常地有感覺...

  • 覺得少了什麼時看 Code: The Hidden Language of Computer Hardware and Software (via 宅色夫)
  • Coursera Programming Languages
    • 上完這個就會覺得戰哪個語言比較好很蠢。
    • 上完這個後會覺得寫OOP很蠢。
