gov到g0v，零時政府對政治有什麼想像？希望帶來哪些改變?

我對於政治的想像，希望它是專業且理性、開放的。就像一個優秀的產品，除了本身技術卓越，仍積極吸納使用者的想法，嘗試創造更好的產品。政治影響著我們的生活，它的一舉一動都要讓我們理解、接納，反應。

更開放、更透明、更容易參與，更民主。我們選出來的代表更能代表實際民意。

面對更多影響深遠的政策議題，運作過程也更透明。

問：原本的政治是不開放的嗎？或者進入門檻高？

我認為兩者都有。許多爭議的議題，一般民眾不上街頭就沒有話語權，有話語權的政治人物則隱藏在黨的指揮下不願論述，於是政策的推動或監督都由抽象又無法對話的「黨」來執行。而地方政治的運作情況更加不透明，除了經常與地方民代博感情之外，幾乎沒有管道對地方政治有所了解，回應。現有的開放資料，或受限於技術，或受限於規劃，並無法讓更多人理解到政治的現況與變化，有意願者往往不得其門而入。

民意代表不再代表民意，而是代表黨意、財團意，政治越來越多由上而下的執意而為，背離民主價值，透過資料的透明與開放，希望能稍微增加一點施壓監督力道。現在政府面臨的問題比以往更複雜，未來政府可能需要更多的群眾參與，而不只由少數政府官員決定。利用資訊科技，這一點逐漸變成可能。（陸續在國際上也看到這種發展趨勢）

問：因此參與者希望透過open data，讓更多人知道政治的運作?

這是其中一部分，有了理解才能有合切的反應。而 Open Data 還存有更多的可能性，政府釋出可用的資料，後續的應用往往會超越原始的想像。

為什麼做 open data? 

(open data vs liberating data)

其實是希望解決問題，一直以來目的並不只是「想做」open data。當我們遇到的問題是不透明的資訊，自然就開始想想辦法讓資訊透明，進而發現政府的 open data 尚未啟動，整體環境也不友善（資料開放量、格式、授權都不優）（還記得實價登錄的新聞嗎？）。所以g0v第一年作很多挖資料、整理資料的 liberate data 的工作。先把資料解放出來，制定 API ，同時也不斷要求政府 open data，透過已有的成果施壓，目前已經透過這樣的模式刺激政府本身產出「真正的 open data」。g0v的先行，也是希望民間政府在 open data 政策上有一個溝通基礎，促成政府資料透明。日後政府開放資料愈多，g0v不需要再花力氣爬資料整理資料，就可把能量轉往應用層面，提供更多好的服務。

  • 註：open data 一般定義為政府主動將資料開放出來，讓社會再利用應用。g0v雖名為零時「政府」，但還是民間 XD

目前有哪些背景的成員 (NGO, NPO, 獨立評論, 公民記者? )，如何分工？

  • hacker、工程師、設計師、插畫、文字、影像工作者

分工有幾種方式。一種是以專案分，各自組隊參加自己有興趣的專案，看需要什麼幫忙直接開幹。另一種是以專業分，大致有分為程式組、文化部（文字、插畫、音樂、設計等非程式組）。

kao參與方式完全自由選擇參加，目前有「人力資源部」（右側歡迎跳坑處）http://hack.g0v.tw/project，讓各專案把待辦事項列出，有興趣的人就可以去撿事情來作。也正在規劃更方便的介面，讓人、專案、待辦事項之間能更自動化，更分散即可執行。實體媒合分工就靠兩個月一次黑客松或期間的小型黑客松工作坊。

也就是說 g0v 並無限定參與成員，只要心中有個念頭在蠢動，「你」也可以直接跳下來一同動手作。
