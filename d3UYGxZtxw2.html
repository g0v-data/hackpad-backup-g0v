自然人資訊驗證平台

  • 發起人/拋磚人： Stranger_1980
  • 狀態： 發想中
  • 專案簡介：
  • 從前無網路時，接收資訊的管道是老師、權威人士、父母鄰居、親朋好友、報紙、圖書館、專業書籍、電視、收音機等單向媒體。
  • 目前是網路時代，接收資訊的主流管道是網路媒體、搜尋引擎。
  • 網路時代，資訊量比以前多很多，目前已演變到真假難辯、甚至有垃圾資訊、網軍、寫手大量做資訊操作，有越演越烈趨勢，許多人不僅身受其害，無形中也造成媒體的公信力降低。 
  • 有鑑於此，在未來，勢必要有辦法解決網路資訊公信力降低的問題。
  • 網路應該是目前散佈知識最有效率的媒介，它的工作應該是將正確的資訊做傳遞，做正向教化而非反向教化。
  • 因此這個提案是利用網路的力量，透過數量極多的自然人去修正資訊的公信力。

  •  主要選項為，相信、懷疑、不知道
  • http://ppt.cc/-LdH@.jpg

  •   透過這些選項的選擇，自然累積各選項的人數，達成新聞的公信力求證。

  • 道德評價為：20/100      
  • 商戰評價為90/100
  • 代表此資訊在商戰上有極高評價，可是道德評價為最低。

台灣公民百科

  • 發起人：貓橘毛
  • 狀態：建構中，http://citizen.wiki.g0v.tw/ 
  • 專案簡介：維基有mediawiki可以做平台，所以有很多類似wikipedia的網站如a+醫學百科等，主要的想法是將現有的社會議題（如大埔案、服貿等）寫入，藉由wiki可多人共編的特性，讓關心議題的人能自由加入各新聞、網路資料的連結和分享頁面資料（簡單來說單一頁面就像是完整的懶人包）
  • 工作目標：散播、記錄公民應關心的社會議題。
    • 詳細：事件需詳細記錄起因、過程、爭議、政府作法、媒體報導等。新聞頁面需求證來源(如源自於哪個平面媒體或是電子報、報導人是誰…等)。
    • 確實：資訊需有公正的來源性(如媒體、照片、影音等證明)，FB及論壇等個人言論應不予以記錄。
    • 爭議：事件必定會有兩方以上的說法，需詳列出各方(政府(含立委，需區分黨籍)、媒體(各大報)、鄉民(?)、被爭議者、法介人士及專業人士…等)的說法；基於詳細原則，需讓頁面的查詢者能如觀看懶人包的方式打那些搞不清楚狀況的人的臉XDDD
    • 備份：避免原始連結資料遺失、特定立場人士說詞反覆、媒體報導偷改已刊登的資料，必須將相關資料以「備份」的主題頁寫入百科內。但請務必確實原則，不得濫用備份功能，將個人的言論寫入百科備份中，予以抨擊。（立委、官員的言論視為代表人民發言，故得予以備份，讓公民能夠監督其作為。)
  • 授權方式：同wikipedia授權(CC-BY-SA)
  • 使用資料：網路/平面上允許公開傳播之相關資料。(他人文章請徵求作者同意)
  • 徵求夥伴（有興趣直接簽名）：
    • NeedsWriter: 需要文案幫手（撰寫基本資訊、報導專案 etc）
    • NeedsDesigner: 需要介面設計
    • NeedsData: 需要資料（擷取、清理）
    • NeedsTech: 需要技術支援（程式、架站 etc)
    • NeedsProcess: 需要幫忙設計作業流程
    • NeedsTalkingToRealPerson: 需要有人幫忙和其他機關聯絡
    • 根據這些議題的重要性，有一些應該不違反wikipedia的規範，是不是直接使用wikipedia就可以了呢？以功能、目標與授權而言，兩者乍看之下大致相容。當然有個獨立的公民百科就可以不用受到wikipedia規範的限制，比如說備份功能，在wikipedia上可能會被退版。
    • 沿用原wikipedia的功能會有中立性的問題 (見: wiki大埔事件 )，核心構想寫在工作目標下的四點，希望可以詳細、確實的記錄事件雙方有爭議的論述、媒體的吹捧和煽動抹黑染色，並備份相關資料。
    • 這樣後續也可以拿出來當歷史資料做為參考。(公民議題包含政府的運作，所以各立委在立委的言論、提案等都是可以被記錄進去的)  希望有回答到你的問題~
    • 使用mediawiki / wikilike的主要問題是，是否要登入才能夠編輯。這是一個網站是否能夠活下去的雙面刃~
    • mediawiki的頁面權限是可以設定的(在安裝的時候有選項是關於這個，也有允許未登入就可編輯的選項)，註冊的資料很簡單只要填帳密跟信箱就可以了；當然也有想過要讓使用者直接一鍵註冊(透過fb之類的)，不過畢竟是早上才弄出來的東西也沒那麼快XDDD
    • 原 wikipedia 也是有一群熱血的維基人在做編輯和維護的動作~依鄉民的個性，呃，應該說依台灣人熱心的民族性，我想也是會有人願意看到事件就動手整理，然後編進去吧XD(當然前提是這百科有廣為流傳啦...)

新聞小幫手

by Ronny Wang (對不起我真的不會想名字，所以只會取 xx 小幫手)

==想法==

新聞小幫手是一個瀏覽器外掛，裝上去之後，如果你正在看的新聞有被人指出錯誤，或是有另一面向的說法值得一看，就會浮出相關連結提示你，幫新聞做好平衡報導。

另外使用者也可以選擇新聞小幫手記錄下你最近N天的看過哪些新聞，若是你N天內看過的新聞被人提出有錯誤或是另一角度的說法，或者結合News-Diff的新聞被修改記錄，會跳出通知讓你知道。

由於現在新聞為了時間壓力，很容易未經查證錯誤百出，或者為了收視率壓力，用了聳動卻錯誤資訊。而現代人也是比較忙碌，往往沒有時間對所有新聞一一查證，所以新聞小幫手要幫大家記住大家最近看過哪些新聞，如果新聞有錯就主動通知你，讓你不要不小心把錯誤的訊息也記在腦袋中了。

==可能可以顯示的東西==

  • 有人的說法跟其他人甚至自己的矛盾, 就附上連結
  • 新聞報導網路上的事喜歡不附連結, 網友自己附
  • 有些國際新聞、外電報導附上原文網址，有能力的人可以幫忙看是否誤譯或斷章取義
  • 科學報導(ex英國研究)加上 paper link
  • 天天在傳的謠言自動加上網路追追追的 link
  • 程式自動 match 相似的同一則新聞, 方便比對各家說法
  • 利用 news-diff 的資料顯示 changelog

從想法延伸的UI和功能，目前以功能性和簡易性為主：

  • 新聞網頁可以讓每個網友對每一句話進行討論跟標記錯誤，有點類似highlight或是像wiki連結的功能。
  • Highlight的部份基本上是隱藏的，只有滑鼠移過去可以顯示討論或是錯誤。
  • 除了錯誤標籤外，可以加上過度煽情等其他等標籤。
  • 在新聞標題下有一個整體性的統計，可以讓閱讀者快速知道是否值得看，其中可以包含：
    • 正確性，統計這篇文章錯誤多寡。
    • 討論性，熱門程度。
    • 對照的新聞，或是直接提供g0v.tv討論議題的連結。
    • 記者/作者的正確性，爭議性統計，或是他的熱門，高正確性新聞。
  • 這個部份比較不急，可能可以有專業新聞閱讀者，以防熱門議題的新聞有一堆人胡亂標記錯誤部份。

== 呈現方式 ==

  • 類似求職小幫手，最下面跳出黃色提示告訴你這則新聞被人提出的錯誤在哪
  • Facebook 裡面假如有人分享新聞，就自動在塗鴨牆的摘要的部份顯示錯誤在哪
  • 記錄下使用者最近看過的新聞，若是他看完之後才被人指出錯誤的，就用類似 MSN 跳出訊息的方式(HTML5 Notification) 告訴他
