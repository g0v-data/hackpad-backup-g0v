2016 中選會選舉資料意見回饋

API意見回饋

  1. 沒有候選人唯一識別碼
  2. API中無法分辨全國不分區和僑居國外國民兩選區，API連結
  3. 欄位名稱不固定
  • 例如政見，在全國不分區是politics；區域或原住民都是rptpolitics，雖然說全國不分區的是政黨統一政見，意義上有些不同，在此提出意見供討論
  1. 異體字
  • 彰化縣第二選區黄玉芬，黄 vs 黃，也許是候選人自己選擇的，但是中選會登記概況時釋出的資料是正體字
  1. 無候選人照片
  2. 出生地是否需要臺灣省
  • 臺灣省彰化縣 vs 彰化縣
  1. api 介面命名過於草率，目前是用 1~8 作為命名，也許該思考所有資料納入後的介面應該長什麼樣子去設計
  2. 選舉區對應之行政區，如果已經是該鄉鎮市區所有村里應該列舉鄉鎮市區即可，每個都列出村里會有些多餘
  3. 投開票所應該直接附上 WGS84 地理座標，理想上應該還有該投開票所涵蓋行政區或 polygon 座標
  4. 公辦政見會資料應該一律錄影並且提供檔案位置（或 youtube 連結）
  5. 各種代碼應該進一步在文件提供完整代碼表的參考資料
  6. 應該可以直接連結並且提供監察院政治獻金專戶的資料共同呈現
  7. 實際產生選舉公報後希望能夠比照 2014 提供電子檔，並且將連結加入各候選人 api 資料中
  8. 跟第一點一樣希望有唯一識別碼，並且希望依據這個識別碼能夠新增一個介面去查詢該候選人過去參選記錄，因為跨年度的資訊直接用姓名連結的錯誤很多
  9. 回應的 http header 希望能夠加入 Access-Control-Allow-Origin: * 等聲明，讓外部應用可以直接讀取(ref: https://developer.mozilla.org/zh-TW/docs/HTTP/Access_control_CORS  )
  10. 個別資料希望加入版本號、建立日期與異動日期
  11. 可以考慮提供申請，讓外部引用各種資料的網站或 app 可以將網址等資訊提供給中選會做交叉連結
