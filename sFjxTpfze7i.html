ＯＯ，這件事您怎麼看？/ 自己的公投自己辦

發起人/拋磚人：noracami

  • 狀態： 概念發想
  • 專案簡介：利用身分證、健保卡驗證之類的具有可靠性的身份驗證機制，提出問題，得到之統計資料以 open data 形式供其他應用。
  • 口號:「自己的公投自己辦」??
    1. 關於投票人註冊
      1. 需通過身份驗證，如 戶政司 國民身份證領補換資料查詢作業 https://www.ris.gov.tw/zh_TW/webapply/6
      2. 但我用自己的資料查是失敗的，不曉得是不是ie限定
      3. 健保卡用一般讀卡機，在公開資料區有以下資料:
      4. 健保卡ID、身份証字號(全碼)、中文姓名、性別、出生年月日、發卡日期

      6. 只要寫一個簡單的 java applet 就可以抓出來，而且全平台(Windows + linux + OSX)可用，
      7. 但健保局提供的 HCA(Healthcare CA) SDK，並不適用健保卡(只能用在醫事卡以上)，更不用說啟動健保卡的PKCS#11功能(其實有沒有我也不知道)，也因此我們無法利用健保卡進行數位簽章。所以這邊會需要一個可靠的機制向 server 傳送這些資料並且驗証資料是來自於我們的 java applet (防止造假)

      9. 如果是自然人憑証的話，有提供相關的數位簽章SDK，但是申請人必需是政府機關、組織團體或是公司行號。但是自然人憑証的發卡量目前約300多萬張，和健保卡95%左右的普及率還是有差。

      11. 最後傳送這些個資的時候，也必需考慮到個資法的問題
      12. 以身分證驗證為例，必填欄位為 身分證字號、發證日期、發證地點
      13. 其他必填欄位可為：年齡、性別等。
      14. 選填欄位供會員註冊完自行填寫，如居住地、職業等。
    2. 關於問題
      1. 由需求單位提供，經過 peer review ，整理成客觀、不預設立場之問題（題組）。
      2. 分級程度可調整 如 １～５分、１～１０分。
  • 這邊提供另一個想法，問卷/民調/投票(whatever)，提出後，只要有一定的人員(EX: 50-100人，門檻可調整)願意使用實名機制連署，就可以通過了。這樣子可以減少人力審核的問題。另外可以考慮加入事後的檢舉機制。特殊情形下可由管理者手動bypass連署期，但必需列明在網頁上做為記錄。
    1. 關於回答
      1. 可決定公佈哪些個人資訊，對於單個問題或者全部問題。
      2. 可改變自己的答案（看法會依時間改變）。
      3. 統計結果可以時間為單位觀察意見比例之變化。
  • 可以考慮增加選項，統計資料是否為實時(real time)反應在網頁上。或是只有於投票結束後才能觀看。有些種類的民調/投票/問卷，讓使用者看到現在的投票趨勢，可能會造成一定程度不準確的影響。
  • 此專案目標希望以真實性、準確性為原則，提供研究人員有效參考。
