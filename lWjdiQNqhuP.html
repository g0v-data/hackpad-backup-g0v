零時政府與阿龜的邂垢 G0V x Agrithon@Taiwan

為什麼要做這個

G0V 這邊光整理農業相關的就一百多個坑了，常常可以看到大家整理了一些列表以及分類，但說真的... 坑坑相連到天邊，究竟怎麼個連法其實隔坑如隔山(!?)，光列表家分類似乎不是什麼有 sense 的事情... (被毆飛)

所以在想的是，是不是再來挖一個坑，打算引用文字分析的技術，然後分析後轉為網路關聯圖，用視覺的方式串聯目前 G0V 與各種農業相關議題，以及農業黑客松的關係。順利的話，這份網路圖除了有視覺的效果外，應該也能夠在這基礎之上順手進行分群 (graph-based clustering) ，應該就可以把概念相近的坑送做堆了 (遠目)。

想要達成的目標

希望能讓所有想要挖坑填坑跳坑的人，都能很輕鬆地了解相關聯的坑有哪些，促進社群相關坑的串聯及活絡

預計作法

爬完所有 G0V Hackpad 中有關農業的資料 => 斷詞及詞性標註(CKIP) => 關鍵詞萃取(POS Pattern + TF-IDF) => 議題相似度計算(Cosine Similarity) => 社會網路分析(Weighted-degree Centrality, Modularity clustering) => 視覺化 (Gephi/D3/Vis/Sigma)

  • 藉由社群更新維護一份與農業相關坑的清單
  • 網路爬蟲爬進每個坑把資料 dump 下來
  • 斷詞處理 (tokenization)
  • 自然語言分析及關鍵詞分析
  • 字詞共現分析
  • 坑坑相連到天邊的共現網路視覺化結果 (1-mode, homogeneous network)
  • Web 端連結 (JSON/GEXF to D3/Vis/Sigma)
  • 目前輸出資料直接用 Vis 的 ForceAtlas2 排好像有點難收斂跑不完, 接下來改從 Gephi 輸出排好的 JSON/GEXF 直接用 Sigma 畫畫看
  • 自動化上面的流程

需要的資料

  • G0V 農業相關坑的清單 (G0V@Hackpad)

需要的人力或資源

  • 網路爬蟲
  • 中文斷詞系統 (CKIP)
  • 資料視覺化 (Vis.js or D3.js)
  • 前端框架 (Boostrap)

N+1 視覺化結果

  • 節點：G0V 中與農業議題相關的 118 個坑
  • 連結：坑與坑彼此之間的議題相似度，越粗代表相似度越高
  • 顏色：坑與坑之間相似度結構所凝聚而成的議題群集，不同顏色代表不同的議題
  • 大小：越熱門的坑節點看起來會越大 (連結量+議題相似度)
