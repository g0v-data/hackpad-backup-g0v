一般認為geek 是阿宅 是很自我中心的，這些人怎麼會關心公共事務

Geek 在資訊領域可以解釋為「技術狂熱者」或「技術宅」，但在台日流行文化裡的「宅」可能衍生自動漫次文化，兩者有本質上的不同，阿宅百百款，一概而論容易出錯。g0v 的 geek 是一群對寫程式有熱誠的人，對資訊掌握度也高，不滿足於現下傳統媒體反覆放送的單一價值觀的 geek，「宅」在家裡可能是在瀏覽器的 40 個分頁裡吸收大量資訊。因為從網路接收的資訊太多了，反而更容易生氣，有了能將怒氣化為行動的社群，能量就很可觀。

原本 open source 就有許多社群，大家彼此都有輕度連結，要聚集起來把討論技術的熱情轉到公共事務相對少了一點組織門檻。有了良好的正向循環，就有更多不同類型的阿宅加入，ptt宅、動畫宅、純粹宅（只是愛上網）。（在網路分眾時代，其實大家都很宅吧）

其實我覺得常常（宅在家裡）上網的人反而更容易關心公眾事務，因為資訊很流通，接收各方的多元資訊，對公民意識的培養是很有幫助的。就我自己來說，上網多年後整個思維有大幅改變，原本我也覺得政治很麻煩，只想逃避這些討厭的事情，只想趕快得到一個結論去把政治現象合理化，好讓自己可以置身事外。但在螢幕後面，我可以安全地觀賞（？）各種論戰，邊看邊在心裡揣摩各種想法，也被網路謠言騙過好幾次……經過這些網路世界的紛紛擾擾後，人會逐漸地習慣接受挑戰，習慣改變想法，開始充滿懷疑。上網多年的過程中，我逐漸從自我中心的人轉變成熱衷公共事務的人。相較之下，只活在三次元世界的朋友則是幾乎數十年如一日。至於 geek 跟自我中心的關連？（有很多世界知名的 geek 都很熱血啊，linus 什麼的…） 我猜，thomas 心目中的 geek 形象可能跟高功能自閉症混在一起，所以會有自我中心的印象？在我認識的人裡面，反而越 geek 的人越是熱血。(像 Aaron Swartz也很熱血）

g0v用什麼機制，讓這些專業人才持續參與

 

  • 線上聊天、github開源、即時共筆？和++機制？XDDDDD
  • 自己做的事看起來似乎有機會改變現狀。

工具

  • IRC

平常利用線上聊天室 IRC（Internet Relay Chat）溝通，目前 24 小時掛在線上的大約兩百多人，也有聊天紀錄可以查閱，不少人雖然沒有直接貢獻 g0v 專案，但都持續有在關心 g0v 話題。社群生活化、貢獻也就容易在聊天中自然產出。

身為一個非資訊圈出身的正常人，我是參與 g0v 後好幾個月才開始經常性地使用 irc 聊天室（主要是申請了 irccloud 以後），結果不用則已，一用沈迷 XD irc 的即時性，讓我感覺隨時都跟這些人在一起，德不孤必有鄰也～不過 irc 固然好用，但社群最重要的還是人，合得來的人加上可以輕易和這些人保持聯絡的工具，結果就是每天都能有靈魂共鳴，精神層面感到非常充實。

  • 線上即時共筆、mailing list

所有的點子、想法、計畫都在線上公開討論，完整 archIve，即使專案擱置或暫時無人響應，但過一陣子有人看到資料就可銜接。除了寫程式可以分工，連寫演講稿、採訪稿都可以多人分工，不僅速度快也互補，也在累積共同完成任務的團隊感。

說到 hackpad、hackfoldr 這些共筆工具，我覺得跟非資訊社群相比，（開源）資訊社群在運作上多出很大一截優勢，那就是因為所有的文件和程式碼都攤開在網路上，所以任何人想加入專案開發、或者接力前人的成果，幾乎都不需要通過「找高層/核心人物橋事」這道手續（儘管大家還是會習慣性地打聲招呼，但那是純粹社交性質，而不是加入專案開發的必要條件，因為所有必要的步驟 github 都幫你做好了），只要能上網，就能看到所有資料，只要會用 git，就能加入程式開發。換句話說，這個人坑介面是標準化的、不受任何人壟斷的。剛開始不覺得這有什麼稀奇，實際參與一陣子後，我才體會到少掉「找人橋事」這道手續竟然會讓進入門檻降低這麼多。在一般的工作場合，我習慣安分守己、不在其位不謀其政、事不關己不過問，以免被認為多管閒事、引起別人的反感。但在一個資訊完全透明、公開近用的環境裡，我可以光明正大地閱讀別人的文件而不會被認為別有居心，既然都看得到，順手幫忙編輯補充也就顯得順理成章。公開的線上共筆讓所有人少掉一大截心裡障礙，很顯著的結果就是參與變得非常踴躍，很多根本不知打哪來的野生路人都會跳下來兩肋插刀一番，社群的成長也變得異常迅速。

社群氣氛

很多團體組織開會發言的起手式可能是先批評別人沒做到什麼，或是別人作不好，但 g0v 鼓勵大家可以分析問題，但一定要提出自己想執行的方案。提出方案、提出需要的幫助，邀請大家參與。如果成果是可以開放分享（開放原始碼、開放授權），就會容易讓事情轉動起來。

 以我個人來說，讓我願意持續參與的原因有：

  1. 沒有客戶、上司管我，不用跟誰橋事、不用看誰臉色，愛做什麼就做什麼，很爽，做出自己想要的東西很有成就感。沒人欣賞無妨，有人喜歡更好，重要的是自己覺得好玩
  2. 做到一半的東西會有人來幫忙，覺得很溫暖，不用寒窗十年（？）地默默自幹
  3. 可以一直學到新的技術，變得越來越厲害
  4. 可以跟聰明幽默的頂尖程式人共事，有趣又開心
  5. 即使偶爾遇到不喜歡的人事物，反正這是社群又不是工作，不理會就好啦，也沒人能拿我怎樣 XD shit doesn’t happen!

回想起來，這些讓人沈迷的要素，幾乎都是 open source 機制底下會自動衍生出來的。其實 open source 圈內的人通常有幾項很鮮明的共同特質或作風：

  • 少說話多做事 - 因為已經有很方便的工具（git），有意見的人要嘛就開 issue，要嘛就送 patch，因此在一般世界中常發生的「袖手旁觀說涼話」現象在開源碼的世界裡並不常見，反正程式碼都公開在那邊，有什麼意見的話，有本事就自己拿去改，沒本事就閉嘴。換句話說，開源機制會自然而然地孕育出一種 doer 文化，鄙視嘴砲、鼓勵捲袖子，久而久之，就會吸引越來越多 doer 加入，這些樂於付出的人會彼此扶持，產生一種滾雪球似的正面循環。
  • 100% 真本事 - 沽名釣譽之徒在這裡沒有生存空間，因為哪個人寫了哪幾行程式碼都清清楚楚，每個人的實力在哪裡（對看得懂程式碼的人來說）一望即知，資訊完全公開，幾乎沒有人能藉由操作資訊不對等來獲取好處。在開源碼的圈子裡，只有真正值得受到敬重的人會享譽盛名，在人際上獲得回饋。這形成了一套近乎理想的獎勵系統，社群內的資源（通常就等於聲譽）會往真正有貢獻的人身上流動，於是誘使所有人努力鍛鍊真本事、做出貢獻，這使得社群的產能比一般商業專案高出數倍（其實產能高有多重原因，但我認為這是其中一個）。
