Fork, Customize and Deploy

Step 0: Getting Started

You will need the following to proceed:

  1. a domain name
  2. a github account

Step 1: Fork Hackfoldr 2.0 on GitHub

  1. login to https://github.com/
  2. go to https://github.com/hackfoldr/hackfoldr-2.0-forkme
  3. click the fork button at top right corner

Step 2: Edit CNAME file in gh-pages branch

  1. go to the shiny new github repo you just created by forking
  2. click the branch button at left top area and select "gh-pages"
  3. click the CNAME file at root directory
  4. click the pencil button at right top area to edit the CNAME file
  5. put your domain in the CNAME file
    • make sure the CNAME file contains one single line only
    • example: https://github.com/moztw/hackfoldr-moztw/blob/gh-pages/CNAME

Step 3: Set Up Your DNS

Take Amazon Router 53 for example:

  • name: _YOUR_DOMAIN_NAME_
    • e.g. folder.moztw.org
  • type: CNAME
  • value: _YOUR_GITHUB_ACCOUNT_.github.io
    • e.g. moztw.github.io
  • evaluate target: -
  • health check id: -
  • ttl: 300

Or take Cloudflare for example:

  • CNAME: _YOUR_GITHUB_ACCOUNT_.github.io
    • e.g. etblue.github.io
  • Page Rule: http://etblue.github.io/+ forward到 http://etblue.github.io

Reference: Cloudflare Guide http://blog.cloudflare.com/introducing-pagerules-url-forwarding/

    在 master branch 開發時使用自動 deploy

    在 master 開發完後需要 deploy 到 gh-pages 才會生效。

    deploy 的方法有兩種：

    1. 在 master 產生出 .html .css 後手動下 git 指令 merge 到 gh-pages（it works，但不推薦）
    2. 讓程式自動幫我們跑 deploy 指令（it works smart，推薦）

    兩種 deploy 方式中，自動 deploy 又分兩種：

    1. 半自動 deploy script
    2. 全自動 travis ci
