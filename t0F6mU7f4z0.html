議題關注者

  • 20字小標
    •

    長期關注議題的 NGO 工作者、公僕、律師等各領域專家。

  • 150-250 內文
    •

    我長期關注議題 (政治、社會、環境、性別、人權等等)，熟悉議題的發展脈絡、可能的資料來源、以及議題的爭議點。我所關心議題需要好的數位工具來幫助議題討論或宣傳，所以我可以：

    1. 來 g0v 黑客松提案 (參考：如何提案)。
    2. 運用專業知識，幫助現有的專案編寫文案、簡報、影片、網宣等媒材。
    3. 幫專案成果寫深入報導，搭建社群與大眾的橋樑。
    4. 我還可以透過以下方式參與 g0v：
      • 在 g0v hub 上用關鍵字尋找有興趣的專案
      • 在 g0v search 上尋找並閱讀的文件

    設計師

  • 20字小標
    •

    關心使用者體驗的感受，在乎介面美不美、好不好用，希望打造 g0v 風格的你。

  • 150-250 內文
    •

    找到喜歡的專案後，我可以：

    1. 幫 g0v 設計 CIS，客製化 g0v 風格的 bootstrap / jqueryUI 等等元件。
    2. 參與製作專案的 UI/UX  或 mockup ，提供現有設計的改進建議。
    3. 釐清使用族群、規劃使用流程，重新設計使用者體驗。
    4. 分析資料，提供資料最簡單的視覺化設計。
    5. 我還可以透過以下方式參與 g0v：
      • 在 g0v hub 上用關鍵字尋找有興趣的專案
      • 在 g0v search 上尋找並閱讀的文件
      • 在 g0v github 上尋找解決設計相關 issue

    工程師

  • 20字小標
    •

    專案開發、視覺呈現---所有專案都需要你！

    150-250 內文

    我可以瀏覽各專案的程式開發需求，依照自己擅長的開發語言在以下管道尋找貢獻的地方：

    • 在 g0v hub 上用關鍵字尋找有興趣的專案  
    • 在 g0v search 上尋找並閱讀的文件
    • 在 g0v github 上尋找可解的 issue

    如果我是開放源碼社群的新朋友，可以參考：  

    • release early, release often: 常常更新最新狀況，讓所有人都能用你寫好的東西，避免重複的工作
    • 儘早把工作項目切小，並找人幫  
    • 認識開源之道 簡報, 字稿  
    • 如何與開放源碼社群工作

    文字工作者

  • 20字小標
    •

    專案介紹、翻譯、新聞寫作---每個專案都需要用文字說故事的你

  • 150-250 內文
    •
    • 我可以在了解各專案的用途後，做這些事：
    1. 撰寫各專案的介紹、網站文案、預計目標等，讓更多人知道專案的功用與人力需求。
    2. 翻譯有關開放資料、公民參與等任何我認為對 g0v 有幫助的文件，例如之前「開放議會宣言」導論就是靠大家協力翻譯完成的。
    3. 寫資料新聞可以找 g0v 內的黑客幫忙協作動態視覺化作搭配 (ex. Kirby 畫的捷運人數成長動態圖)，從資料爬梳出圖文並茂的精彩觀點。
    4. 將自己參與黑客松的心得、對 g0v 的想法或採訪的寫作發表成客觀的報導，讓更多人明白 g0v 的運作。
    • 我可以在以下地方跟其他參與者討論：
      • 在 g0v hub 上用關鍵字尋找有興趣的專案
      • 在 g0v search 上尋找並閱讀的文件
