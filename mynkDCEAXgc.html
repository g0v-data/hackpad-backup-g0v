鄉民關心你 - 專案介紹

什麼是鄉民關心你(Kuansim)

一個企圖處理蒐集，整理，串連，行動，追蹤的去中心化群眾參與社會議題的平台。

是串接事件和人的媒介，是對話的一個窗口，它存在的目的，最基礎是備忘，其次是串聯和關注，更積極的是通往後續的「行動」，整套都接通了，才是鄉民關心你的全貌。

  • 有沒有考慮要加入所謂帶狀責任的概念進去??因為是追蹤各種事件或政策延續的發展，其實也可以清楚去回溯到久遠以前，某件"政績"或"事件"是橫跨世代、政黨輪替、好幾個官員或原本就是政務官一般性工作的內容，一直持續累積發生的(或是遠因)，其責任(或政績)不是只有某個人應該100%全有的，鄉民關心你或許也可以突現出"帶狀責任"的概念
  • 責任歸屬這個部分要先討論出個明確的範圍才有辦法做帶狀責任，像是關廠工人事件這到底該算誰的責任

名詞定義

  • 事件(Event): 有時序的資料，可以是新聞、公文、社群網路討論串、公聽會等等。
  • 標籤(Tag): 資料集的屬性。
  • 分類(Category): 互斥的資料集 。
  • 立場(Position): 對特定議題的觀點。
  • 問題(Problem): 通常由行政官僚，政策分析師等提出對策
  • 議題(Issue): 政客，媒體，政治顧運透過分析、界定、區格與標籤等手段將問題轉化成“對社會真實的爭辯。 p.123, 公關政治學
  • "問題是必須加以解決的，而議題卻只是政治人物用來混淆選民以追求本身仕途的工具" ~ Dirty Politics, Kathleen H. Jamieson
  • 議題設定: 指大眾傳播媒介通過報導內容的方向及數量，對一個議題進行強調。
  • 議題框架效應: 一個問題用兩種說法，兩種說法在邏輯意義上其實是相似的，然而卻導致了不同的決策判斷。
  • 期望結果: 群眾判斷要不要對特定議題繼續爭辯的準則。
  • 信任網路(Trust Network):  http://en.wikipedia.org/wiki/Wikipedia:Trust_network
  • Filter Bubble: a result state in which a website algorithm selectively guesses what information a user would like to see based on information about the user (such as location, past click behaviour and search history) and, as a result, users become separated from information that disagrees with their viewpoints, effectively isolating them in their own cultural or ideological bubbles.

目標與功能

想解決的問題

  1. 議題相關資訊取得，檢驗的成本太高, 使群眾難以掌握真相。
  2. 針對議題採取行動的聯繫成本太高，使群眾難以改變現狀。
  3. 沒有一套機制使群眾可以追蹤他所關心過的議題。
  4. 群眾該關心的議題被電視，電台，報紙決定，而非群眾本身。

會是怎麼樣的網站，怎麼完成上述的目標

  1. 為了解決問題1, 群眾需要議題文獻整理及檢驗工具以產生議題摘要(懶人包)
  2. 為了解決問題2, 群眾需要一種串連機制，能將匯聚起來相似立場的民氣, 轉化成實際的行動，解決爭議。
  3. 為了解決問題3, 群眾需要一種追蹤的機制, 使得社會議題在失去主流媒體興趣後, 仍能持續回顧議題的最新狀態，以採取下一步，直至爭辯雙方達成共識 (Close the loop)。
  4. 為了解決問題4, 群眾需要建立Trust Network以篩選想接收的資訊，但這帶來另一個問題(Filter bubble)，需要找個一個折衷方案。
  5. 為了降低發言被管制的可能，這個平台必須是分散式的。

尚未解決的問題

  1. filter bubble
  2. garbage information

發展方向(Road map)

  • 階段一:  降低群眾整理議題脈絡的時間成本
  • 階段二:  降低群眾針對議題採取行動的聯繫成本

預定功能

這玩意大概是這幾種東西的複合體

  • 書籤 : 整理議題相關文獻
    • 產生時間軸
    • 產生人物關係圖
    • 產生地裡關係圖
  • 日記 : 紀錄對特定議題當下的想法, 因始可以回顧自己看法的轉變
  • 通訊錄 : 找到正確的人
    •  政府公開通訊錄
  • 糾團 : 找戰友 
    • 找到相似立場的人
    • 找到相似立場的活動
  • 任務追蹤 : 提案或表達意見後看事情有沒有改善
  • 成就系統 : 增加鄉民參與公眾事務的動力
    • warring: 量化指標容易引導公眾只關心數據而非問題原因
  • 報表: 看視覺化數據
    • warring: 量化指標容易引導公眾只關心數據而非問題原因
  • API

使用方式

  • 蒐集事件, 評論
    • 社交網路討論串 (fb, twitter, g+, plurk, youtube, etc)
    • 新聞報導
    • 可利用 LFTT 做自動分類 Use https://ifttt.com/wtf
    • 自動關聯自己社交網路上有tag的討論串
  • 分類
    • 加標籤，標籤是種名稱，名稱只是物件的指謂
    • 寫意見
      • 針對標籤(可能是對象)
      • 針對對象
      • 針對事件
  • 定期回顧
    • 回顧自己收集的事件
      • by 人. e.x 014編年史
      • by tag. e.x 腦殘
    • 回顧相關事件(別人蒐集的事件)
      • 針對事件/標籤寫意見
  • 向目標表達意見
    • 目標為組織或個人
    • 請願/開幹 (delegate to other web site)
      • 加入別人或自己來
      • 書面表達 (傳真 or Email, 對方網站)
      • templat system, and guiding doc. (怎麼寫交戰守則)
      • 行動表達
    • 倡議
      • delegate to other web site
    • 讓網路社群意見進入中央/地方政府輿情會報。（台南市是目前唯一將「社群新聞」正式納入輿情會報的單位，橋接線上討論與現實公務員體系的有效方法，電話有效、地方單位的1999專線也會繞過公文體系直接呈報市長辦公室，或直接交辦處理。）
