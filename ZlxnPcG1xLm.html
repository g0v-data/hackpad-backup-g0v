動民主 x 公民憲政會議 = 動憲法

衍生自 動民主 lab 人體試驗 #2

上層：憲餅廚房

背景

  • 動民主 2.0 規劃卡關後，經一番研究發現，網路公民參與的關鍵並不在於資訊系統，而在於建立審議的 SOP 以及培養多位能夠梳理議題的主持人，資訊系統必須依附在這個基礎上才能發揮作用。詳見這篇 google+ post。
  • 目前公民憲政會議推動聯盟（憲動盟）內有審議民主強者坐鎮，正在進行一系列活動，也就是處理上述的基礎部分，但礙於人力，無法將公民論壇收集到的珍貴資訊完善地數位化。

主旨

  • 中程：以憲動盟建立的真槍實彈的公民論壇資料為基礎，打造能夠輔助 NGOs 進行公民憲政會議的網路平台，以促進 2016 憲法公投為第一優先目標。
  • 遠程：把上述網路平台帶到其他審議民主計畫做測試，將它調整成可以廣泛適用於各種情境的版本，然後回頭更新動民主 2.0，以便導入到 NGO 以外的地方，例如政府單位舉辦的公民論壇中。

空污場資料整理，問題：

  • 前言、專家導讀的部分好像應該要以某種簡潔的形式呈現在摘要版的會議紀錄中，這些背景資訊好像蠻重要的
    • solution: 全紀錄，除了問題、提案之外新增「導讀」分類，呈現時再用 UI 的 filter 設定為預設隱藏。這樣整份紀錄的資料格式還是可以維持統一
  • 有些句子講到一半就不見了是…？？ XD
    • 有原始錄影可以參考嗎？
  • 有些錯字，e.g. 憲動盟打成線動萌之類的，已逕改
  • 向 brad 解釋用 google spreadsheet 當後台的運作原理
    • brad 表示這是對一般人來說都非常 ok 的編輯介面，可以讓各場紀錄 follow 這個格式
    • meta data 可以直接在同一個檔案內另開一個 sheet 供憲動盟編輯
    • 潛在風險：
      • 紀錄本身就要去識別化，不能記發言人真名 -> 可能要有雙紀錄機制（逐字稿紀錄 + 手勢反應紀錄），搭配名牌
      • 因為 gsheet 資料會即時更新到網頁上，紀錄上線後表單編輯權限要鎖起來，以免誤改
      • 頁面全部都在瀏覽器內即時 render，舊 device 可能會跑得慢，另可能會對 SEO 不利，要測試過才知道
        • SEO: 暫時先手塞內文到隱藏 div，有需要的話一兩個月更新之類的
        • vTaiwan 的做法是每分鐘從後端跑 google-spreadsheet-download 再 commit 到 gh-pages 裡（與之前 g0v.today 定期讀取 ethercalc 的做法相同），或可參考。
    • 潛在 todo：
      • 憲動盟自己開 github org
  • 論點可以連到白板照片，白板照片也可以回到論點
    • 實作方式可以加一個隱藏標籤（其實應該是資料表欄位 XD）標註連到哪張照片
    • 更正：應該是意見書要對應到原始紀錄，就像是寫論文要有註腳一樣
  • 論點多一個「依意見書結構排序」的選項
