2.  我要抗議 //議題

  • 原因：動議來源於抗議，抗議的聲音有時候並不夠大. 有了抗議，人民就會提出議題。當然這個議題還可以針對企業：如某某企業加班很嚴重這樣.
  • 身分驗證問題
  • 好玩的東西：能不能有個好玩的界面呢？例如用戶自己設計抗議的動作以及動畫這樣？
    • 提供 表情符號？ 基本動作？
  • 類似網站：https://petitions.whitehouse.gov/ （做得很好看！）
  • 看到這個突然想到，如果不是「蒐集新聞」而是「製造新聞」呢？民眾提出各類議題讓各位鄉民投票，票數高的就拿出來討論，就像 ptt 的八卦版一樣，但是更有組織，不會像ptt 那樣資訊較為零散。問題來了，那應該怎麼推廣以及搶ptt 族群？

  • 解決方法：social network api

議題提出的思考:

應該怎樣提出？

  • 問題是什麼？eg. 反服務貿易黑箱
  • 提出這個問題的理由是什麼？
    • 如對公民利益損害大
    • 30秒過服務貿易太不合理
  • 解決的辦法是什麼？
    • 可以嘗試提出：例如公開透明
    • 不過解決辦法可能涉及多方意見，以及多方面的知識，大家意見又不一致，這個時候怎麼辦？
      • 全民共同起草一份文件，這樣可以集聚所有人的意見
        • 如果有爭議該怎麼辦？
  • 我想加入一個投票的環節. （因為馬英九說國內外的人都支持服貿嘛）
    • 原因：如果很多人都反對這個議題，它是否需要解決呢？
      • 不過反對的理由是什麼呢？
    • 可是怎麼樣才能夠讓大家上網來投票呢？
      • 如果重要的議題將會得到政府的關注會不會好一點呢？
      • http://mag.udn.com/mag/news/mag_votelist.jsp 上面的投票人數很少. 
  • tag標簽
    • 可以對每個議題都加一個tag標簽（由市民自己加），然後每一個標簽下面都能看到大家對這一類議題最關心的是哪個？
  • 覺得可以有不只一個標籤~~~~~(想標什麼就標什麼)
    • 增加曝光度&連結到更多相似討論網頁，比如說使用者幫議題加了"學運"，學運標籤二字可以附上超連結，連到https://www.facebook.com/hashtag/%E5%AD%B8%E9%81%8B?source=feed_text&story_id=763495536993822 (FB的#學運page)或是其他地方。不過後來想想，這樣其實好像沒有增加到曝光率....
    • 可能要想一下tag 是不是要有限制？否則人人都加一個tag 的話反而會變得很亂。其實我覺得目前的新聞都會有tag, 是否可以透過撈資料的方式直接把各個新聞的 tag 抓出來就好？
  • 好像比較可行，不過還是希望使用者能夠有選擇的機會，那麼或許把相關新聞的tag抓出來做為選項讓他們勾一個？
