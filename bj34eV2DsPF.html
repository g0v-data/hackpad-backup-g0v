g0v 小學校 / levelup

Introduction

建立類似 diy.org 成就/任務/徽章系統的概念, 以循序漸進 + 由簡入繁 + 指令型態的項目，讓新鮮人漸漸了解 g0v 的目的, 並且讓更多的人進入到"主動" "參與" 的情境。

預計是 g0vis 視覺化呈現的一部分：g0vis 希望提供很容易讓人理解的 g0v 訊息, 而 levelup 則是試著引導人參與其口，但系統可以用 API 的形式獨立出來，讓各專案都可以利用。

  • 新手教學文是大坑（寫 code, 寫 help documents, 兜機）

徽章

成就歷程

  • 透過已解任務描述角色狀態. 例如
    • 已解 nodejs + sass + angular + ... ==> 全端工程師 get
  • 個人成就屬性 ==> 法師(設計師) 戰士(工程師) etc.. 由 star graph 之類的作分析後給予稱號
    • 搞清楚屬性後, 就更方便找人 或 推坑了

成就/任務，例如:

  • 參與活動
    • 找到一個有興趣的議題, 並在 idea pool 描述
    • 黑客松報名成功
    • 黑客松吃飯時間找到三個不認識的人
      • 了解他們在進行的專案
    • 參與一個三人以上的專案
      • 並且名字被記載在 g0v.json contributor 中
    • 發起一個專案
      • 找到兩個貢獻者
      • 建立粉絲頁
        • 獲得 1000 個讚
      • 登上新聞
      • 與政府單位對話 (視專案類型而定)
      • 獲得(任何)獎項
    • 推坑
    • 短講
    • 進入演講坑
    • 發起迷你松
  • 參與開發
    • commit 到 open source project
    • 發 pull request 給自己並未直接參與的專案
    • 為專案設計 api 並釋出
