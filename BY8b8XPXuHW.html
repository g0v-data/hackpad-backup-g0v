主旨

這次黑客松的主題中，news pac / people watch / 鄉民關心你 / 化憤池 / 我的失憶政府 / 社會議題懶人包 / 社會議題追蹤網 / 這幾個專案對於新聞抓取、tag 的部分是類似的，但各自面對的族群和使用情境也不是完全一樣。做成產品給人使用的時候，可能會有收集資料（新聞、PTT、blog評論） / 用大家收集的資料製作懶人包 / 找出某議題的相關單位來施壓或抵制 / 腦力激盪提出政策性解法... 等種種用途。

基礎的議題 meta 應該可以跟我們 g0v project hub 一樣，訂出一份大家都能用的規格建一個共用的資料庫，至於前端的應用方式就各專案各自發揮。明天可能請大家先把自己想到的用途所需要的 meta 欄位開一開之類的.... （沒概念中）

clkao: 要開始整理一個共用元件區嗎？譬如說 auth, 也許就可以共用，到時候如果 g0v user 可以作 sms verification 甚至數位簽章驗證 就可以直接 sign 過去
