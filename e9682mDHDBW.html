g0v grant - CMS for news

# § 請以 80 ~ 120 字簡短地說明這個專案

Dippen(沾水筆) + Space(空間) = DipSpace(暫定)。空間裡有一罐墨水，任何人拿起文具，沾墨水都可以提筆寫字，這是一個以協作為核心理念出發的CMS專案。

從「單向傳輸」進入「雙向互動」的時代，我們希望讓每一個閱讀者的貢獻都有價值。過去會在每一篇文章最下方有零散的  commit 這樣消極的對話並不符合我們需求。

# § 這個計畫要解決什麼問題？

我們希望讓每一個閱讀者的貢獻都有價值，並讓其他閱讀者可以從一篇文章/報導上：

- 看到更多元的面向

- 更簡便的方式對於內容的細項議題可深入了解

- 讓文章/報導不會因為語言差異而造成屏障

# § 你為什麼要做這個計劃 （ 個人動機 ）？

對 無物為真、協作參與、多元聲音 有所期待。

# § 你預計用什麼方式解決此問題？

WYSIWYG Markdown Editor https://yabwe.github.io/medium-editor/ (fork 改)

資料使用 Markdown 儲存有助於資料維護，透過所見即所得的介面讓編輯者不需要學會 Markdown 語法即可知道發佈後的版面，閱讀者也可以透過此相同界面在閱讀時提出修改/補充。

jsdiff 文章版本比對 https://github.com/kpdecker/jsdiff

透過 diff 機制可以比對修改前後的差異，有助於 pull request 流程。

資料庫將以行為儲存單位，有助於多人協作及多國語言翻譯。

類似 CMS 的開源專案, 任何媒體拿到 source code 都可以自行架設一個網站。

# § 打算如何讓社群參與以及回饋意見？

1. 將 sketch 一併放置於 repository 內協助開發.

2. G0v slack 會有 #DipSpace 可供即時的討論.

3. 每兩週一次的線上小松(2hr), 線上coding跟大家分享進度並收集意見.

4. Repository 會放在 github g0v/DipSpace 上, 因此 issue / pull request 依舊.

5. 看板方式 https://help.github.com/articles/about-projects/ 協調工作進度, user story 也會在上面.

6. 任何人都可以在 https://productpains.com/product/dipspace 許願, 票數高的可能就會是下個 release 的目標
